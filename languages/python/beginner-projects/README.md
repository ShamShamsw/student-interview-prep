# Beginner Projects (01-50)

Hands-on Python projects that now span from absolute beginner scripting to intermediate simulation and visualization workflows.

This folder started with six planning-focused projects and has been expanded to a full 50-project sequence with richer multi-file architectures, reproducible outputs, benchmarking, and plots.

## What Changed

- The collection now includes projects 01 through 50.
- Many projects now generate visual artifacts (charts, timelines, confusion matrices, trend plots, and image panels).
- Intermediate projects use a consistent multi-file pattern: `main.py`, `models.py`, `operations.py`, `display.py`, and `storage.py`.
- Later projects focus on reproducibility with deterministic seeds, persistent run history, and exported JSON artifacts.

## Project Tracks

| Track | Project Range | Focus |
|---|---|---|
| Foundation | 01-06 | Core Python control flow, functions, data structures, file I/O, and documentation habits |
| Applied ML/CV/NLP demos | 07-20 | Practical beginner-friendly machine learning, image/audio/text pipelines, and visual analysis |
| Data and systems tools | 21-35 | Labeling, search, crawling, optimization, chemistry/graph/text workflows |
| Interactive simulators | 36-47 | Annotation, anomaly detection, recommender systems, collaboration, weather/sign language/sorting/physics |
| Benchmark-first demos | 48-50 | Runtime analytics, persistent histories, strategy comparison, and exportable artifacts |

## Recommended Learning Path

1. Complete 01-06 in order for fundamentals and code organization.
2. Pick any 2-3 projects from 07-20 based on your interests (NLP, CV, or data exploration).
3. Move to 21-35 to practice broader workflows and data tooling.
4. Use 36-47 for simulation-heavy and visualization-heavy practice.
5. Finish with 48-50 to learn benchmarking, trace export, and reproducible experiment reporting.

## Definition Of Done

A completed project should include:

- A clean run of the main entry point without crashes.
- Clear terminal output and readable summaries.
- Persisted outputs where supported (JSON run files and chart/image artifacts).
- Documentation updates in that project's README.
- Consistent structure and naming aligned with the existing project pattern.
